## Problem Statement

Remote GraphRAG generation failed during node embedding:

```text
Get embedding of nodes: 32709/32713
[ERROR][Exception]: Operation timed out after 10800 seconds and 1 attempts.
```

The deployment uses a private embedding endpoint. The likely root cause is not simple network reachability. The more important issue is that GraphRAG can produce tens of thousands of nodes/edges, while the current implementation sends many single-item embedding calls and can exceed the total task timeout.

## Required Implementation

### 1. Batch embedding

Implement batch embedding for GraphRAG nodes and edges.

Requirements:

- Batch node texts before calling `embd_mdl.encode`.
- Batch edge texts before calling `embd_mdl.encode`.
- Keep output mapping deterministic.
- Support configurable batch size.

Suggested default:

```text
GRAPHRAG_EMBED_BATCH_SIZE=16
```

### 2. Dedicated embedding concurrency limiter

Do not depend only on `MAX_CONCURRENT_CHATS`.

Requirements:

- Add a GraphRAG embedding-specific limiter.
- Default concurrency should be conservative.
- Make it configurable.

Suggested default:

```text
GRAPHRAG_EMBED_CONCURRENCY=2
```

### 3. Bounded internal queue

Do not create one async task per node/edge for large graphs.

Requirements:

- Process batches through a bounded queue or bounded worker pool.
- Preserve ordered mapping.
- Report completed item and batch progress.

### 4. Retry and backoff

Embedding batches must retry transient failures.

Requirements:

- Bounded retry attempts.
- Exponential backoff with jitter.
- Log batch index, attempt, batch size, elapsed time, and failure reason.
- Permanent failures must stop the run safely and leave it resumable.

Suggested defaults:

```text
GRAPHRAG_EMBED_MAX_RETRIES=3
GRAPHRAG_EMBED_RETRY_BASE_SECONDS=2
GRAPHRAG_EMBED_RETRY_MAX_SECONDS=60
```

### 5. Configurable timeout

Make the 3-hour task timeout configurable.

Requirements:

- Add environment-controlled timeout for GraphRAG or task execution.
- Do not rely on timeout increase as the only fix.

Suggested variables:

```text
RAGFLOW_TASK_TIMEOUT_SECONDS=10800
GRAPHRAG_TASK_TIMEOUT_SECONDS=21600
```

### 6. Large-graph fault tolerance

Requirements:

- Preserve old graph until all new graph/vector chunks are prepared.
- If a batch fails after all retries, mark task interrupted/resumable.
- Keep enough failure context in progress logs for operator action.
- Integrate with `resume_failed` behavior where practical.
