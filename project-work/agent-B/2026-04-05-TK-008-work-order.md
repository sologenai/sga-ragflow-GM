## Background

`TK-006` has already made graph evidence visible in chat, and the product value is confirmed:

- users can now see graph participation
- entity and relation evidence is useful and well received
- community summaries are valuable when present

However, the current graph-evidence card gives community summaries too much visual weight.
Long summaries can dominate the entire message area and make the evidence panel feel like a second article rather than a compact supporting component.

This follow-up ticket is not about retrieval correctness.
It is a presentation and usability refinement ticket.

## Goal

Keep community summaries available, but make them feel controlled and expandable instead of overwhelming by default.

The desired user experience is:

1. community summary is still visible
2. full text is still available
3. default state is compact
4. user can explicitly expand to read the full summary
5. entities and relations remain easy to scan

## Required Outcomes

### 1. Community summary must be collapsible

The graph-evidence panel must support a collapsed default state for community-summary text.

Expected behavior:

- long summary content is truncated or visually folded by default
- a clear UI action allows users to expand and read the full text
- users can collapse it again if appropriate

### 2. Do not remove the summary

This ticket must not hide or delete the community summary feature.

Not allowed:

- removing community summary content entirely
- moving the only summary access into a hard-to-find modal
- replacing the summary with a placeholder that loses substance

### 3. Keep entities and relations readable

The current entity/relation value is important and should remain visually accessible.

Preferred direction:

- community summary stays compact by default
- entities and relations remain visible without requiring a large scroll

### 4. Keep scope small

This is a UI refinement ticket, not a retrieval redesign.

Allowed:

- a compact collapsible component
- a small “expand / collapse” interaction
- light visual hierarchy improvements inside the graph-evidence block

Not allowed:

- major chat layout redesign
- graph-evidence data contract redesign
- backend retrieval changes unless a tiny UI-supporting field is strictly necessary

## Suggested Implementation Direction

Recommended direction:

1. keep the graph-evidence card
2. render community-summary content in a collapsed container by default
3. add a clear action such as:
   - `展开全文`
   - `收起`
4. preserve full summary text when expanded
5. keep entities/relations in the same card, easy to discover

Optional polish if low risk:

- clamp default height or line count
- add a subtle fade mask in collapsed state
- keep the expand control close to the summary block, not at the card footer
