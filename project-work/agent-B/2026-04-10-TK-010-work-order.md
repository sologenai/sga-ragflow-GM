## Background

The project previously had a GraphRAG resume capability.

The current codebase now shows a mismatch:

- `api/apps/kb_app.py` still calls resume-related task-monitor APIs
- `rag/graphrag/task_monitor.py` in current source no longer implements the enhanced compatibility surface expected by those calls

This mismatch breaks main-service startup in the GPU image and also means the GraphRAG resume feature is no longer reliably present in source.

Reference document:

- `project-work/agent-x/2026-04-10-graphrag-resume-merge-mismatch-analysis.md`

## Goal

Restore the GraphRAG resume-support implementation so that:

1. the main service can start normally in the GPU image
2. GraphRAG resume/trace behavior is no longer source-mismatched
3. the restored implementation aligns with the existing `kb_app.py` usage instead of removing that functionality
