# 2026-04-04 Project Status

## 项目当前定位

本项目以 RAGFlow 0.24.0 为底座，已经演进为一套“企业知识平台 + 智能体工作台 + 内部系统同步能力”的综合项目，而不是单一的文档问答系统。

## 当前核心模块

- `api/`：主后端服务，负责 API、鉴权、任务编排、同步入口。
- `rag/`：RAG、GraphRAG、检索、LLM 适配与处理流水线。
- `deepdoc/`：OCR、版面分析、表格识别、复杂文档解析。
- `agent/`：智能体工作流、工具调用、插件、沙箱执行。
- `web/`：前端工作台，覆盖 dataset、search、chat、agent、memory、mcp、admin。
- `memory/`：智能体记忆能力。
- `mcp/`：MCP 服务接入能力。

## 已识别的业务化定制

- 新闻同步：已具备后端调度、接口、管理页面、图谱重建触发。
- 档案同步：已具备接口、分类映射、管理页面、手动触发，但自动调度成熟度低于新闻同步。
- 数据源同步：保留了通用 connector 能力，可对接外部平台。

## 当前项目判断

- 这是“平台 + 客户专项定制”混合态项目。
- 平台能力已经较完整，但客户化逻辑已经深度进入主干代码。
- 后续若继续扩展，需要严格管理工单、验收和职责边界。

## 当前主要风险

- 上游升级成本会逐步升高。
- 部分业务系统接入信息存在硬编码风险，需要后续治理。
- 新闻与档案同步成熟度不一致，交付与运维标准需要统一。
- 业务专项能力自动化测试覆盖不足。

## 接下来建议

- 按工单制推进所有修改。
- 项目经理只做分派、测试、验收、归档，不直接承担功能开发。
- 每个执行 agent 独立记录自己的工单过程和验收结果。
