## Problem Statement

Remote deployment has already configured a `qwen3-vl-embedding` model, but the model cannot be used reliably.

The likely root cause is vector-dimension incompatibility. RAGFlow stores embeddings under fields named by vector length, for example `q_1024_vec` or `q_4096_vec`. If the document-engine mapping does not support the returned dimension, parsing or retrieval can fail even though model configuration succeeds.

The immediate suspected case is a 4096-dimensional Qwen embedding model used against an Elasticsearch mapping that currently supports only 512, 768, 1024, and 1536.

- `text-embedding-3-large`: 3072 dimensions by default.
