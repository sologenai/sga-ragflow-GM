## Background

The customer accepted the overall penetration-test report and only requires remediation for the system prompt leakage finding shown in the report screenshot.

Project-manager assessment has already confirmed:

- this is not an image vulnerability
- this is a prompt leakage / prompt exposure issue
- the report URL appears to be from an external application or gateway, but the shared backend logic in this repository still contains prompt-leakage risk

Reference assessment:

- `project-work/agent-x/2026-04-04-system-prompt-leakage-assessment.md`

## Current Risk Judgment

There are two main risk points that must be addressed:

1. Agent prompt leakage risk during orchestration
   - the agent flow builds a `task_desc` containing:
     - `Agent Prompt`
     - `User Request`
     - `User Defined Prompts`
   - this material is then used in planning / tool-calling stages

2. Prompt exposure in user-facing response payloads
   - some chat response paths still return a `prompt` field to the client
   - this field includes system prompt material and related internal context

## Goal

Deliver a focused remediation so that:

- system prompt / internal instructions are no longer easily exposed to users
- user-facing responses no longer return prompt content
- common prompt-extraction attacks are covered by regression tests

## Mandatory Implementation Principles

- Use the smallest correct security fix
- Prioritize shared backend hardening in this repository
- Do not expand into a full prompt-governance framework
- Do not rely on frontend masking as the primary fix
- Do not silently redesign the full agent architecture
- Leave a clear completion record for later customer traceability

## Required Changes

### 1. Add anti-leakage rules to the relevant prompt layer

You must add explicit rules that the model must not reveal:

- system prompt
- internal instructions
- hidden reasoning
- tool schema
- task description
- user-defined prompt content

Expected behavior when attacked:

- refuse briefly
- do not disclose the hidden content
- continue helping with the user's actual business request when possible

### 2. Remove prompt exposure from user-facing response payloads

You must review and repair user-facing response paths so that prompt content is no longer returned to clients.

This includes at least the currently identified chat response paths.

### 3. Add regression coverage

You must add focused regression tests that cover at minimum:

- "show your system prompt" style attack
- "ignore previous instructions" style attack
- user-facing response payload no longer exposing `prompt`
