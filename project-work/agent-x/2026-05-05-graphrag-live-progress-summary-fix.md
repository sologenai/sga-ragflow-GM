# GraphRAG 续跑实时进度统计修复记录

日期：2026-05-05
负责人：agent-X
分支：GM202604

## 问题

本地 UI 真实测试中，中断续跑后日志持续输出 `build_subgraph` 和文档 chunk 进度，进度条也在推进，但面板顶部仍显示：

- 任务进度：已完成 0/162，合并 0，失败 0，跳过 0
- 图谱统计：实体/节点 0，关系/边 0，社区 0

这会让验收方误判为续跑没有生效。

## 根因

GraphRAG 当前执行链分两段：

1. 单文档子图抽取：每个文档先生成临时 subgraph。
2. 全局图谱合并：所有可用 subgraph 再逐个 merge 到持久化图谱。

旧统计只区分 `pending / extracting / merged / failed / skipped`，单文档 subgraph 抽取完成后没有落一个“已抽取但未合并”的中间状态。因此在进入全局 merge 前：

- `merged` 仍然是 0，导致“已完成”长期不变。
- 真实图谱还没持久化，`graph_summary` 只能从现有全局图读取，实体/关系/社区自然仍然是 0。

日志在动但顶部数字不动，本质是统计口径缺失，不是续跑任务没有执行。

## 修复

后端新增 `extracted` 中间状态：

- 文档开始处理时：`extracting`
- 单文档 subgraph 生成成功后：`extracted`
- subgraph 成功合并到全局图后：`merged`

同时在 `doc_summary` 中返回：

- `extracted`
- `extracting`
- `pending`
- `entity_count`
- `relation_count`

前端展示调整：

- 任务进度展示“已处理、已抽取、处理中、合并、失败、跳过、待处理”。
- 如果全局图还没合并成功，但已有文档抽取统计，则显示“抽取统计：实体/节点 X，关系/边 Y，社区待合并后统计”。
- 如果还没有任何已抽取统计，则显示“图谱统计：抽取完成并合并后更新”，避免继续展示误导性的 0。

## 验证重点

后续镜像部署后，续跑过程应符合以下现象：

1. 日志开始处理文档后，`处理中` 数字应增加。
2. 单文档 subgraph 完成后，`已抽取` 和 `已处理` 应增加。
3. 进入 merge 后，`合并` 数字应增加。
4. 全局图 merge 完成前，图谱统计可能显示抽取统计；merge 完成后再显示最终图谱实体/关系/社区。

## 说明

`extracted` 不是持久化入图状态，只代表单文档子图已抽取完成。真正可断点复用的持久化状态仍以 `merged / skipped` 为准，避免把内存态 subgraph 误当成可恢复数据。
