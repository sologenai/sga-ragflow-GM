## Rework Background

Live browser verification shows that the account-management work from `TK-007` is still incomplete for ordinary users.

Current verified runtime behavior:

- superuser self-service password change in the RAGFlow profile page works
- ordinary users can still encounter a silent failure when trying to change their own password from the same profile page
- the visible symptom is: clicking `Save` appears to have no effect, with no clear success or failure feedback

Reference review:

- `project-work/agent-x/2026-04-05-TK-007-review-round-1.md`

## Rework Scope

### 1. Fix the ordinary-user self-service password dialog

This is required.

The password-change modal must no longer depend on unrelated required profile fields such as name/timezone in a way that blocks password submission.

Expected outcome:

- password dialog validation only enforces the fields relevant to password change
- hidden or unrelated profile fields do not silently block submission
- clicking `Save` either submits successfully or clearly reports a user-visible reason for failure

Primary files to inspect:

- `web/src/pages/user-setting/profile/index.tsx`
- `web/src/pages/user-setting/profile/hooks/use-profile.ts`
- `web/src/hooks/use-user-setting-request.tsx`

### 2. Preserve the already-corrected backend password convention

This rework must not regress the code-path corrections already made in:

- `admin/server/services.py`
- `api/apps/user_app.py`
- `admin/server/auth.py`
- `api/db/services/user_service.py`

Expected outcome:

- admin reset path still works
- self-service path still sends the intended password payload to the backend
- no regression is introduced into the `hash(plain password)` convention

### 3. Improve user-visible feedback for self-service password change

This is required.

Expected outcome:

- successful self-service password change clearly closes or updates the modal
- failed self-service password change clearly surfaces validation or backend feedback
- the user is not left in a no-response state after clicking `Save`
