## 1. 外部影像调用接口

### 1.1 元数据查询接口
> 元数据查询接口不区分权限，所有数据均可查询（仅元数据）

#### 1.1.1 调用鉴权
接口调用包含请求头 `RequestHeader`：

- systemid（必填，string） — 业务系统 id
- token（必填，string） — 令牌

示例 Headers：

```
systemid: OA
token: 6e39b0181cfba5637a1b03daa19d2070
```

测试环境获取 token：

```
http://das-dev.itg.cn/report/getApiToken?systemid=XX&password=xx
```

参数：
- systemid：业务系统 id（需先由档案系统配置）
- password：密码

token 有效期：24 小时

成功返回示例：

```json
{"result":"success","Token":"6e39b0181cfba5637a1b03daa19d2070"}
```

失败返回示例：

```json
{"result":"false","message":"systemid或密码错误"}
```

OA 接口调用配置示例：

- systemid: 
- password: 

---

#### 1.1.2 查询入参信息

地址：

```
http://das-dev.itg.cn/report/queryDataDetail
```

请求方式：POST

说明：入参信息非必填字段支持模糊查询、多条件查询。

常用入参字段（说明、是否必填）：

- sysno (String(32)) 必填：用于记录查询系统及校验该系统是否注册，可调用数据，如 OA、SAP
- doctype (String(32)) 非必填：查询类型，如 GD、SFD 等
- userid (String(32)) 非必填：记录查询用户，员工号
- docclassfy (String(32)) 非必填：档案分类号，增加查询效率
- docbarcode (String(128)) 非必填：文件的条码、二维码编号
- oso （docno）(String(128)) 非必填：文件编号，如关单号、发票号、合同号等
- doctitle (String(512)) 非必填：文件标题（支持模糊）
- doccomptaxno (String(128)) 非必填：公司纳税人识别号
- archivetypename (String(128)) 非必填：门类中文名称（如：凭证档案）
- isfilepool (String(8)) 非必填：默认 false，不填/空均为 false；为 true 时限定只在【文件库】全宗范围下查询档案
- filetypename (String(128)) 非必填：文件库门类（isfilepool 为 true 时生效）

示例查询 JSON：

```json
{
  "sysno": "SAP",
  "doctype": "SFD",
  "userid": "308569",
  "docclassfy": "10",
  "docbarcode": "DA-DM-0000001002",
  "doctitle": "进口增值税"
}
```

#### 1.1.3 返回明细参数
返回参数根据文档类型可配置，`SysInfo`、`DocInfo` 为固定出参，`MetaData`、`ProInfo` 可自定义。

##### 1.1.3.1 来源系统信息 SysInfo

字段说明（示例）：

- sysno (str(32)) — 系统编号，如 OA、SAP
- billtype (str(32)) — 业务系统约定类型
- billno (str(32)) — 单据号

##### 1.1.3.2 文档信息集合 DocInfo

常见字段：

- docid (str(32)) — 当前业务系统记录的唯一 ID
- docfid (str(32)) — 文档父 ID
- doctype (str(8)) — 文档类型（如 HT、FP）
- docdate (int(8)) — 文档日期，格式 yyyyMMdd
- docyear (int(4)) — 文档年度（可由系统自动截取）
- docform (int(2)) — 文档载体（枚举）
- docclassfy (str(8)) — 文档分类号
- docbarcode (str(64)) — 文档条码
- docno (str(64)) — 文档业务编号
- docversion (int(4)) — 文档版本，默认 1
- doccomptaxno (str(64)) — 文档归档公司纳税人识别号
- doccompname (str(64)) — 文档归档公司名称
- docdeptname (str(64)) — 文档归档部门
- doctitle (str(256)) — 文档标题
- dockeyword (str(256)) — 关键词/摘要
- docsecurity (int(2)) — 档案开放程度，默认 30
- docpages (int(4)) — 附件页数，无法计算默认 1
- docauthorcompname (str(256)) — 文档创建/产生单位
- docauthordeptcode (str(256)) — 创建部门 id 全路径
- docauthordeptname (str(256)) — 创建部门全路径名称
- docauthorgpf, docauthorgpfcode, docauthorpf, docauthorpfcode — 利润中心相关信息
- docauthor (str(256)) — 创建人中文名
- docauthordate (str(32)) — 创建日期 yyyyMMdd
- docdesc (str(512)) — 备注
- tablename (str(32)) — 数据存储表名
- tableid (str(64)) — 数据存储 id

##### 1.1.3.3 元数据集合 MetaData

示例字段（以发票为例）：

- itgtotalamount (str(32)) — 价税合计
- itgamountexcludingtax (str(32)) — 不含税金额
- itgsellercomp (str(64)) — 销售方公司名称
- itgsellercomptaxno (str(64)) — 销售方纳税人识别号
- itgpurchasercomp (str(64)) — 购买方公司名称
- itgpurchasercomptaxno (str(64)) — 购买方纳税人识别号
- itgcheckcode (str(64)) — 发票校验码

##### 1.1.3.4 元数据集合路径 ProInfo

示例字段：

- promainno (str(128)) — 主项目编号
- promainname (str(128)) — 主项目名称
- protno (str(128)) — 项目编号
- protname (str(128)) — 项目名称
- protype (str(64)) — 项目类型
- probusinessmodel (str(64)) — 业务模式

### 1.2 附件影像调用

> 仅限文档开放程度 30 以下的数据进行查询，需要提前申请权限。

#### 1.2.1 调用鉴权
同 1.1.1，使用 `systemid` 与 `token` 请求头。

示例：

```
systemid: OA
token: 6e39b0181cfba5637a1b03daa19d2070
```

#### 1.2.2 调用影像入参

地址：

```
http://das-dev.itg.cn/report/gueryViewData
```

请求方式：POST

入参示例：

- docid (str(32)) 必填：附件 id（查询出参提供）
- tablename (str(32)) 必填：文件存储表名
- tableid (str(64)) 必填：文件存储 id
- userid (str(32)) 必填：用户 id（员工号）
- fileclass (str(32)) 非必填：文件种类，枚举值（ORG、DAS、ATT、APF、MDR）

示例请求 JSON：

```json
{
  "docid": "63518319113",
  "tablename": "f3_16_document",
  "tableid": "1716290860951408640",
  "userid": "308569",
  "fileclass": "ORG"
}
```

#### 1.2.3 返回附件地址

返回字段：

- filepath (str) — OFD 附件地址
- downloadPath (str) — PDF 附件地址

示例返回：

```json
{
  "fileList": [
    {
      "downloadPath": "https://shanghai.aliyuncs.com/download?...",
      "filePath": "http://shanghai.aliyuncs.com/hl...pdf?Expires=...&ossAccessKeyId=...&signature=..."
    }
  ]
}
```

若权限不足：

```json
{"result":"false","message":"权限不足，请前往档案系统发起档案借阅流程或申请系统查询权限范围！"}
```

若无附件：返回相应提示信息。

### 1.4 查询内容单点跳转档案系统检索页

#### 1.4.1 调用鉴权
同 1.1.1。

#### 1.4.2 调用地址

- 测试环境：https://das-dev.itg.cn/report/gueryBycontent
- 生产环境：https://das.itg.cn/report/gueryBycontent

请求方式：GET

必填 query 参数：

- systemid (String) 必填
- token (String) 必填
- ticket (String) 必填 — 域账号加密字符串
- content (String) 必填 — 查询内容

ticket 单点认证成功直接跳转至档案系统登录页。

#### 1.4.3 域账号加密示例（Java）

示例代码为 DES/CBC/PKCS5Padding 加密，输出 hex 格式：

```java
package com.zetcode.util;
import java.util.Base64;
import javax.crypto.Cipher;
import javax.crypto.spec.IvParameterSpec;
import javax.crypto.spec.SecretKeySpec;

public class DesUtil {
    public final static String CHARSET = "UTF-8";
    public final static String ENTRY_METHOD = "DES";
    public final static String CIPHER_METHOD = "DES/CBC/PKCS5Padding";
    private final static String secret_key = "@itg.net";
    private final static String iv = "@itg.net";

    // DES 加密字符串并转为 HEX
    public static String encrypt(String data) throws Exception {
        byte[] keyBytes = secret_key.getBytes(CHARSET);
        byte[] ivBytes = iv.getBytes(CHARSET);
        SecretKeySpec keySpec = new SecretKeySpec(keyBytes, ENTRY_METHOD);
        IvParameterSpec ivSpec = new IvParameterSpec(ivBytes);
        Cipher cipher = Cipher.getInstance(CIPHER_METHOD);
        cipher.init(Cipher.ENCRYPT_MODE, keySpec, ivSpec);
        byte[] cipherText = cipher.doFinal(data.getBytes(CHARSET));
        return byteArrayToHex(cipherText);
    }

    public static String byteArrayToHex(byte[] bytes) {
        StringBuilder sb = new StringBuilder();
        for (byte b : bytes) {
            sb.append(byteToHex(b));
        }
        return sb.toString();
    }

    public static String byteToHex(byte b) {
        int i = b & 0xFF;
        String hex = Integer.toHexString(i);
        if (hex.length() < 2) hex = "0" + hex;
        return hex;
    }

    // DES 解密
    public static String decrypt(String data) {
        try {
            byte[] byteArray = javax.xml.bind.DatatypeConverter.parseHexBinary(data);
            byte[] keyBytes = secret_key.getBytes(CHARSET);
            byte[] ivBytes = iv.getBytes(CHARSET);
            SecretKeySpec keySpec = new SecretKeySpec(keyBytes, ENTRY_METHOD);
            IvParameterSpec ivSpec = new IvParameterSpec(ivBytes);
            Cipher cipher = Cipher.getInstance(CIPHER_METHOD);
            cipher.init(Cipher.DECRYPT_MODE, keySpec, ivSpec);
            return new String(cipher.doFinal(byteArray), "utf-8");
        } catch (Exception e) {
            e.printStackTrace();
            return data;
        }
    }
}
```

### 2.2 附录：传输字段详解

#### 2.2.1 来源系统信息 SysInfo

- SysNo (str(32)) — 系统编号（如：OA、SAP）
- BillType (str(32)) — 流程/业务类型
- BillNo (str(32)) — 单据号

#### 2.2.2 文档信息集合 DocInfo
（与 1.1.3.2 中字段一致）

#### 2.2.3 电子文件集合 Files

字段说明：

- FileClass (str(32)) — 文件种类（ORG、DAS、ATT、APF、MDR）
- FileId (str(32)) — 电子文件唯一 ID
- FileOrder (int(2)) — 顺序
- FileName (str(256)) — 文件名
- FilePath (str(512)) — 文件路径（FilePath 与 FileUrl 必须有其一，FileUrl 优先）
- FileUrl (str(512)) — 文件访问 URL
- FileType (str(8)) — 后缀名（pdf、xls 等）
- FileDesc (str(512)) — 文件描述
- FileMd5 (str(64)) — MD5
- FileSize (str(32)) — 文件大小，单位 kb

#### 2.2.4 相关文件集合 Relations

用于传输项目、凭证等组卷维度的数据以建立关联：

- DocId / Docld (str(32)) — 相关文件业务系统记录的唯一 ID
- DocBarCode (str(64)) — 条码
- DocType (str(8)) — 文档类型
- DocNo (str(64)) — 文档编号
- DocCompTaxno (str(64)) — 文档所属公司纳税人识别号

#### 2.2.5 元数据集合 MetaData

MetaData 为自定义元数据集合，根据业务场景扩展。

#### 2.2.6 元数据路径 MetaLink

- MedaLinkType (str(8)) — xml 或 json
- MedaLinkUrl (str(512)) — 元数据文件 URL（优先）
- MedaLinkPath (str(512)) — 元数据文件存储路径

#### 2.2.7 组卷信息 ProInfo

示例字段：

- EmployeeNum (str(64)) — 员工编码
- Title (str(64)) — 员工姓名 / 主信息标题

---

### 2.3 字段枚举值定义（选摘）

载体形式（docform）：

- 1：纸质原件
- 2：原生电子文件
- 3：复印件或数字化副本

开放程度（docsecurity）：

- 0：全公开
- 10：有用户身份可见
- 20：内部用户-通用
- 30：内部用户-通用（需申请）
- 40：内部用户-经营管理类文件（需申请）
- 50：内部用户-重大事项（需申请）
- 60：内部用户-重大决策（需申请）
- 70：机密（审批可见，需申请）
- 80：秘密（审批可见，需申请）
- 90：绝密（审批可见，需申请）

文件种类：

- ORG：原始文件
- DAS：存档文件
- ATT：附属文件

分类名称 / 分类号：见系统约定（示例：00 党务工作、01 纪检监察、... 26 供应链管理）

## 3 借阅接口标准

### 3.1 利用申请发起

#### 3.1.1 借阅单信息

字段说明（示例）：

- orderid (String(256)) 必填：申请 ID
- title (String(256)) 必填：标题
- purpose (String(1024)) 必填：利用目的
- period (String(2)) 必填：利用期限
- enttype (Integer) 必填：利用方式（8-电子利用，6-实物外借，5-现场借，7-复制外发）
- applymemo (String(1024)) 必填：申请备注
- userdeptname (String(128)) 必填：申请人部门名称（OA 可不用）

- retentionperiod (String(32)) 非必填：保管期限
- security (String(32)) 非必填：密级
- archcode (String(32)) 非必填：档号
- fillingnum (Integer(4)) 非必填：归档份数
- docid (String(64)) 必填：文档 ID
- docfid (String(64)) 非必填：文档父 ID
- doctype (String(64)) 必填：文档类型
- docdate, docyear, docform, docclassfy, docbarcode, docno, docversion, doccomptaxno, doccompname, docdeptname, doctitle, dockeyWord, docsecurity, docpages, docauthorcompname, docauthordeptcode, docauthordeptname, docauthorgpf, docauthorgpfcode, docauthorpf, docauthorpfcode 等字段见前文
- docauthor, docauthordate, docdesc 非必填

### 3.2 利用申请回调

#### 3.2.1 获取 token

同 1.1.1 获取 token 的方式。

#### 3.2.2 接口地址

```
https://das-dev.itg.cn/report/borrowcallback
```

请求方式：POST

请求头需包含：`systemid`、`token`。

#### 3.2.3 审批结果信息

- orderID (Long) 必填：申请 ID
- success (boolean) 必填：审批是否通过（true/false）

#### 3.2.4 审批详情信息（数组）

每项包含：

- nodeinfoname (String) — 审批部门（用户所在部门名称）
- username (String) — 审批人中文名
- status (Integer) — 审批状态（1-提交审批，2-通过，3-退回，4-退回撰写人，5-作废，6-办结，7-改派，8-协助，9-完成协助）
- memo (String) — 审批意见
- stationname (String) 非必填 — 审批人岗位
- usercode (String) 非必填 — 审批人账号（员工号）
- phone (String) 非必填 — 手机
- email (String) 非必填 — 邮箱
- position (String) 非必填 — 职务
- lastmodifiedtime (Date) — 审批时间

#### 3.2.5 档案审批详情信息

- id (String) 必填：申请档案 ID
- status (Integer) 必填：档案权限（0-无权，1-有权）
